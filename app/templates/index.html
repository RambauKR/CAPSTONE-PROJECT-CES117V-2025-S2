Project Groups

Group 1: Class Attendance System using Facial Recognition

Objective: Create a system that marks students’ attendance based on facial recognition.

Key Components:

  • Face Detection and Recognition:
    • Implement face detection using OpenCV or similar library.
    • Use a pre-trained model or train a facial recognition model (e.g., using OpenFace, Dlib, or FaceNet).
    • Students should be able to upload their pictures, which will be stored with their names in a database or folder structure.
  • Attendance Marking:
    • When a student looks into the camera, the system should recognize their face and mark them as "Present" in an Excel file or database.
    • If a face is detected but not recognized, mark the student as "Not registered" but present in the system.
  • Data Storage:
    • Save attendance records with the time and date in an Excel file or CSV.
    • Store uploaded student images in a folder with the corresponding names.

To Implement:

  • Face detection and recognition.
  • Excel file generation to store attendance.
  • A feedback system to notify users about the recognition status (present/registered or not registered).

Group 2: Vehicle License Plate Scanner

Objective: Develop a system to capture vehicle license plates, vehicle color, and make/type in real-time.

Key Components:

  • License Plate Detection:
    • Use object detection (e.g., YOLO) to identify license plates from passing vehicles.
    • Implement Optical Character Recognition (OCR) (e.g., Tesseract) to read the characters on the plates.
  • Vehicle Color and Type Detection:
    • Implement a model to identify the color of the vehicle.
    • Use a pre-trained model or train a custom model to classify the vehicle type (e.g., sedan, truck, etc.).
  • Data Logging:
    • Capture images of passing vehicles and save them in a folder.
    • Record the license plate number, vehicle color, type, and timestamp in a CSV file for future reference.

To Implement:

  • License plate detection and OCR.
  • Color and vehicle type detection.
  • Real-time data logging in a CSV file.

Group 3: Car Counting System

Objective: Develop a system to count vehicles in different lanes and categorize them by type.

Key Components:

  • Lane Separation:
    • Divide the incoming video feed into multiple lanes.
    • Use image processing techniques (e.g., background subtraction or object detection) to identify and count vehicles in each lane.
  • Vehicle Counting:
    • Count the number of vehicles in each lane, as well as categorize them by type (e.g., cars, trucks, etc.).
  • Real-Time Display and Control:
    • Display real-time counts of incoming and outgoing vehicles on a web portal.
    • Add buttons for capturing a picture, recording video, resetting counts, and saving results.
  • Data Storage:
    • Save results (vehicle counts, timestamps, and images) to an Excel file with a timestamp whenever the “Save Results” button is clicked.

To Implement:

  • Lane separation and vehicle counting.
  • Real-time display of counts.
  • Buttons for capturing data, resetting counts, and saving results to an Excel file.

Group 4: Car Parking Slot Monitoring System

Objective: Monitor parking slots and display the availability of each slot in real-time.

Key Components:

  • Parking Slot Detection:
    • Use image processing techniques or object detection models to monitor parking slots.
    • Detect whether a parking slot is occupied or unoccupied.
  • Real-Time Display:
    • Color-code the parking slots in real-time: green for unoccupied and red for occupied.
    • Display the total number of available and occupied parking slots.
    • Categorize vehicles by type (e.g., trucks, vans, etc.).
  • Data Storage:
    • Record the number of occupied and unoccupied slots in a CSV file for future reference.

To Implement:

  • Parking slot detection.
  • Real-time display of slot availability.
  • Data logging (occupied/unoccupied slots, vehicle types, and timestamp).
